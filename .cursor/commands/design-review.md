# /design-review — Request a design review

Act as the **Architect** (see `/architect`).

For the question or proposal provided, produce a design review using the Architect's 6-section format:

1. Problem restatement
2. Options considered (2-4 approaches, pros/cons/complexity/alignment)
3. Recommendation
4. Impact analysis
5. Implementation plan
6. Open questions

Reference `docs/ARCHITECTURE.md` explicitly for every design claim.
Do NOT write implementation code — this is a design artifact.
